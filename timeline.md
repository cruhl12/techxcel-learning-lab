# 8-Week Training Timeline

**Goal:** Be demonstrably ready for the Agentic Solutions Officer role by 22 June 2026.
**Capstone:** Meridian Legal AI — see `capstone/README.md`

---

## Week 1 — Apr 22–28
**Theme:** Foundations — Git, GitHub, Prompt Engineering basics

### Tasks
- [ ] Set up GitHub account (if not done)
- [ ] Connect local git repo to GitHub remote
- [ ] Learn: commit, push, pull, branch, pull request
- [ ] Complete 5 prompt engineering exercises in `projects/02-prompt-engineering/`
- [ ] Write your first SKILL.md using the template (document what you learned about prompting)

### Milestone check
Can you push a change to GitHub without help? Can you write a prompt with role, context, task, format, and constraints defined? Yes to both = Week 1 done.

---

## Week 2 — Apr 29–May 5
**Theme:** Prompt Engineering (advanced) + first API call

### Tasks
- [ ] Study Anthropic's prompt engineering docs (see resources.md)
- [ ] Build a prompt pattern library in `projects/02-prompt-engineering/` — at least 5 reusable patterns
- [ ] Get an Anthropic API key
- [ ] Make your first API call in Python — basic chat completion
- [ ] Commit everything to GitHub

### Milestone check
Do you have a working Python script that calls the Anthropic API and returns a response? Do you have 5 documented prompt patterns you could teach to a client? Yes = Week 2 done.

---

## Week 3 — May 6–12
**Theme:** Multi-provider API integration

### Tasks
- [ ] Get an OpenAI API key
- [ ] Build a script that calls both Anthropic and OpenAI with the same prompt
- [ ] Compare outputs — document the differences in model behaviour
- [ ] Understand: tokens, pricing, rate limits, model selection criteria
- [ ] Write a SKILL.md: "How to choose between AI models for a given task"

### Milestone check
Can you explain to a non-technical client why TechXcel uses multiple AI models instead of just one? Can you demonstrate it with a working script? Yes = Week 3 done.

---

## Week 4 — May 13–19
**Theme:** Azure fundamentals

### Tasks
- [ ] Create a free Azure account (azure.microsoft.com/free)
- [ ] Complete Azure fundamentals learning path on Microsoft Learn (see resources.md)
- [ ] Understand: Azure tenant, subscriptions, resource groups, regions, data residency
- [ ] Explore Azure OpenAI Service — understand how it differs from calling OpenAI directly
- [ ] Write a plain-English explanation of Azure data residency for a non-technical client (1 page)

### Milestone check
Could you explain to a client's board why their data stays in Australia? Could you explain what "your own Azure tenant" means without jargon? Yes = Week 4 done.

## Week 5 — May 20–26
**Theme:** RAG and Knowledge Bases

### Tasks
- [ ] Understand what RAG is and why it matters (read, don't just skim)
- [ ] Build a simple RAG pipeline using Python — ingest a document, embed it, retrieve relevant chunks
- [ ] Use fictional Meridian Legal documents as test data (start creating them)
- [ ] Understand: embeddings, vector stores, chunking, retrieval, context window
- [ ] Write a SKILL.md: "How to build a company knowledge base"

### Milestone check
Can you build a knowledge base from a set of documents and query it with a question? Can you explain what a vector store is to a non-technical person? Yes = Week 5 done.

## Warning Signs (check these weekly)

- Journal hasn't been updated in 3+ days → you're drifting
- Current week's tasks feel vague → re-read the milestone check and work backwards
- You're still on week 1 topics in week 3 → compress, don't abandon — get to the milestone and move
- You're building something not on this list → ask: is this moving me toward the capstone? If no, stop
